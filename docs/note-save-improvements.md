# 笔记保存功能改进说明

## 改进概述

笔记保存功能已完全重构，实现了基于 AI 的智能分类、元数据提取和内容处理。

## 核心改进

### 1. AI 分类到文件夹

- **功能**：自动分析笔记内容，选择最合适的子文件夹保存
- **实现**：读取 Obsidian 目录的文件夹结构，让 AI 根据内容主题选择分类
- **解耦**：分类逻辑独立调用，不与其他功能耦合

### 2. 完整的 Frontmatter

每个笔记文件都包含标准的 Obsidian frontmatter：

```yaml
---
title: 文章标题
source: 原始链接或来源说明
author: 作者（可空）
published: 发布时间（可空）
date created: YYYY-MM-DD
date modified: YYYY-MM-DD HH:mm:ss
tags:
  - 网页内容
  - 人工智能
  - 医疗技术
---
```

### 3. AI 生成标签

- **功能**：根据内容生成 1-3 个精准的语义标签
- **规则**：
  - 使用专业术语和领域词汇
  - 避免过于宽泛的标签
  - 中文，2-4 个字
- **解耦**：标签生成独立调用

### 4. AI 提取元数据

- **功能**：从内容中提取标题、作者、来源、发布时间
- **规则**：
  - 标题必须提供（无法提取则生成描述性标题）
  - 其他字段无法确定时返回 null
  - 不编造信息
- **解耦**：元数据提取独立调用

### 5. 两种内容模式

#### 原文模式（summarize: false）
- 直接保存原始内容
- 适合已经整理好的内容
- 保持原文完整性

#### 总结模式（summarize: true）
- AI 整理成结构化笔记
- 输出格式：
  ```markdown
  ## 摘要
  （2-3 句话概括）

  ## 关键信息
  - 要点1
  - 要点2

  ## 正文
  （结构化内容）

  ## 原文摘录（可选）
  > 重要引用
  ```

## 技术实现

### 文件结构

```
src/sync/
├── journal-sync.js       # 主同步逻辑
└── note-classifier.js    # AI 分类和元数据模块（新增）
```

### AI 调用流程

笔记保存时，按顺序执行 4 个独立的 AI 调用：

1. **提取元数据** → `extractMetadata()`
2. **生成标签** → `generateTags()`
3. **分类到文件夹** → `classifyNote()`
4. **处理内容**（仅总结模式）→ `summarizeContent()`

每个函数独立调用 AI API，互不依赖，实现完全解耦。

### 文件命名

- 使用提取的标题作为文件名
- 自动清理非法字符（`/\:*?"<>|`）
- 限制长度为 100 字符
- 如果文件已存在，添加时间戳避免覆盖

### 日记模式

日记模式保持简单：
- 不进行 AI 处理
- 按日期保存（`YYYY-MM-DD 日记.md`）
- 同一天的内容追加到同一文件
- 使用时间标记（`## HH:mm`）分隔

## 配置要求

需要在 `data/config.json` 中配置 AI：

```json
{
  "ai": {
    "baseUrl": "https://api.openai.com/v1/chat/completions",
    "apiKey": "your-api-key",
    "model": "gpt-4"
  }
}
```

## 前端使用

在笔记标签页，用户可以：

1. 切换"原文/总结"模式（滑块）
2. 输入内容
3. 点击"保存到 Obsidian"

前端会自动传递 `summarize` 参数：

```javascript
const options = {
  summarize: currentType === 'note' && summarizeMode
};
```

## 错误处理

- 如果 AI 调用失败，使用默认值（如默认标签"网页内容"）
- 如果分类失败，保存到根目录
- 如果总结失败，使用原文
- 所有错误都会记录到控制台

## 测试

```bash

```

测试包括：
1. 原文模式保存
2. 总结模式保存
3. 日记保存

## 性能考虑

- 4 个 AI 调用按顺序执行（避免并发限制）
- 每个调用设置 `temperature: 0.3`（提高一致性）
- 内容截断：分类和元数据提取只发送前 1000-1500 字符
- 标签生成发送前 1500 字符
