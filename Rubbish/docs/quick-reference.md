# 笔记保存功能 - 快速参考

## 使用方法

### 通过 Web 界面

1. 访问 http://localhost:3000
2. 切换到"📄 笔记"标签
3. 选择模式：
   - **原文**：直接保存原始内容
   - **总结**：AI 整理成结构化笔记
4. 输入或粘贴内容
5. 点击"💾 保存到 Obsidian"

### 保存结果

笔记会自动：
- ✅ 提取标题、作者、来源、发布时间
- ✅ 生成 1-3 个语义标签
- ✅ AI 分类到合适的文件夹
- ✅ 生成完整的 Obsidian frontmatter
- ✅ 使用标题作为文件名

## 文件位置

- **日记**：`Obsidian目录/YYYY-MM-DD 日记.md`
- **笔记**：`Obsidian目录/{文件夹}/{标题}.md`

## AI 处理步骤

笔记保存时会依次执行：

1. **提取元数据** - 标题、作者、来源、发布时间
2. **生成标签** - 1-3 个精准的语义标签
3. **分类文件夹** - 选择最合适的子文件夹
4. **处理内容** - 总结模式会生成结构化笔记

## 配置检查

确保 `data/config.json` 包含：

```json
{
  "ai": {
    "baseUrl": "https://api.openai.com/v1/chat/completions",
    "apiKey": "your-api-key",
    "model": "gpt-4"
  },
  "diary": {
    "obsidianPath": "/path/to/obsidian/vault"
  }
}
```

## 故障排查

### 问题：笔记保存到根目录而不是子文件夹

**原因**：AI 分类失败或没有合适的文件夹

**解决**：
1. 检查 Obsidian 目录是否有子文件夹
2. 查看服务器日志中的分类结果
3. 确认 AI 配置正确

### 问题：标签都是"网页内容"

**原因**：AI 标签生成失败

**解决**：
1. 检查 AI API 配置
2. 查看服务器日志中的错误信息
3. 确认 API 额度充足

### 问题：总结模式不工作

**原因**：AI 总结调用失败

**解决**：
1. 切换到"原文"模式测试
2. 检查 AI API 是否正常
3. 查看服务器日志

## 日志位置

- **服务器日志**：`logs/server.log`
- **修改记录**：`logs/2026-03-10-note-save-refactor.md`
- **备份文件**：`backups/20260310_222931/`

## 性能说明

- **原文模式**：3 次 AI 调用（元数据、标签、分类）
- **总结模式**：4 次 AI 调用（+ 内容总结）
- **处理时间**：约 5-15 秒（取决于 AI API 速度）

## 注意事项

1. ⚠️ 如果文件名已存在，会自动添加时间戳
2. ⚠️ 文件名会清理非法字符（`/\:*?"<>|`）
3. ⚠️ 文件名最长 100 字符
4. ⚠️ AI 调用失败会使用默认值，不会阻塞保存
5. ✅ 日记模式不受影响，保持原有逻辑
