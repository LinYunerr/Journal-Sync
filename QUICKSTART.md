# 🚀 Journal Sync 2.0 快速开始

## 立即使用

```bash
cd /path/to/Journal-Sync
npm start
```

然后在浏览器打开：**http://localhost:3000**

## 📝 使用流程

1. **选择类型**
   - 点击 📔 日记 或 📄 笔记 标签

2. **输入内容**
   - 在文本框中输入你的内容
   - 支持多行输入和长文本

3. **可选设置**
   - ☑️ 勾选"发送到 Telegram"（会发送 AI 建议）

4. **保存**
   - 点击"💾 保存到 Obsidian"
   - 等待同步完成

5. **查看结果**
   - 查看状态消息（显示各平台同步状态）
   - 在时间线中查看保存的记录
   - 查看统计面板的更新

## 🎯 核心功能

### 日记 vs 笔记

| 类型 | 用途 | 文件名 |
|------|------|--------|
| 📔 日记 | 记录每日生活、感想 | `YYYY-MM-DD 日记.md` |
| 📄 笔记 | 保存重要信息、学习笔记 | `YYYY-MM-DD 笔记.md` |

### 同步平台

- ✅ **Obsidian** - 自动保存到 Markdown 文件
- ✅ **flomo** - 自动同步到 flomo
- ✅ **Nowledge Mem** - 添加到时间线
- ✅ **memU** - AI 提取记忆并生成建议
- 🔵 **Telegram** - 可选发送 AI 建议

### 时间线视图

- 显示最近 20 条记录
- 每条记录显示：
  - 类型（日记/笔记）
  - 时间（刚刚、X分钟前）
  - 内容预览
  - 同步状态

### 统计面板

- 总记录数
- 日记数量
- 笔记数量
- 今日记录数

## 🔧 常见问题

### Q: 端口被占用怎么办？
```bash
PORT=3001 npm start
```

### Q: 如何查看历史记录？
直接在页面下方的时间线中查看，或访问：
```
http://localhost:3000/api/history
```

### Q: Telegram 发送的是什么内容？
发送的是 AI 根据你的历史记忆生成的**智能建议**，不是原文。

建议格式：
```
**相关提醒**
1. 根据之前你和XX的经历...
2. 上次你提到...

**下一步建议**
1. 建议你可以...
2. 考虑到...
```

### Q: 如何停止服务器？
在终端按 `Ctrl+C`

### Q: 数据存储在哪里？
- **Obsidian**: `/path/to/obsidian/notes/`
- **历史记录**: `/path/to/Journal-Sync/data/history.json`

## 📊 API 使用

### 保存内容
```bash
curl -X POST http://localhost:3000/api/save \
  -H "Content-Type: application/json" \
  -d '{
    "content": "今天天气不错",
    "type": "diary",
    "options": {
      "sendToTelegram": true
    }
  }'
```

### 获取历史
```bash
curl http://localhost:3000/api/history?limit=10
```

### 获取统计
```bash
curl http://localhost:3000/api/stats
```
